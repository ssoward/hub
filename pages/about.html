Alma 31 · About

About this study app

Project background, sources, and feedback for the Alma 31 study tool.

What this is

The Alma 31 study app is an interactive companion for one of the Book of Mormon's most instructive chapters. It pairs the full scripture text with thematic analysis, a visual gallery, and practical applications for personal, family, and teaching contexts.

How it's built

The app is a Progressive Web App (PWA) built with plain HTML, CSS, and JavaScript — no framework, no build step. It works offline through a service worker and is installable on phones, tablets, and desktops. The redesigned site shares a single stylesheet across all pages for consistency and accessibility.

  • Markup: Semantic HTML5 with ARIA where appropriate.
  • Styling: CSS custom properties, grid, and a small set of shared components.
  • JavaScript: Vanilla ES2018+ for navigation, filters, lightbox, and search.
  • PWA: Service worker handles offline caching; manifest enables install.

Features

  • Complete Alma 31 text with section markers and search.
  • Deep analysis covering pride, prayer, the power of the word, and compassion.
  • Visual gallery with an accessible lightbox and category filters.
  • Study applications for personal, family, and teaching contexts.
  • Mobile-first, fully responsive, and keyboard-operable.
  • Offline support via service worker.

Sources & attribution

  • Scripture text: The Book of Mormon, public-domain editions.
  • Gallery images: LDS Media Library, public-domain religious artwork, and custom illustrations created for this study app.
  • Analysis: original commentary based on personal scripture study.

Accessibility

The redesigned site targets WCAG 2.1 AA. Every page includes a skip link, visible focus indicators, accessible names on interactive controls, sufficient contrast, and respect for prefers-reduced-motion. If you find an accessibility issue, please open a GitHub issue at the link below.

Feedback

This is a personal project. If you find an error in the scripture text, a broken link, or have a suggestion, please file an issue at github.com/ssoward/hub.
